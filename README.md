InsightX — End-to-End E-Commerce Data Analytics Project
This project simulates a real-world data analytics workflow on an e-commerce dataset using Python and Power BI.

Step 1 — Data Generation (generate_data.py)
Since no real dataset was available, a realistic synthetic e-commerce dataset was created from scratch using NumPy and Python's random module. The script defines 5 product categories namely Electronics, Clothing, Home & Kitchen, Books, and Beauty, each containing 5 products with realistic price ranges. 1,200 unique customers are generated and assigned attributes like region, gender, and age group. For each of the 5,000 orders, the script randomly assigns a customer, product, quantity, discount level, acquisition channel, and order status, then calculates the final revenue after discount. The resulting dataset of 14 columns is saved as ecommerce_data.csv inside the data folder.

Step 2 — Data Cleaning (insightx_analysis.py)
The raw CSV is loaded using Pandas. To simulate real-world data quality issues, approximately 2% of values in the gender and age group columns are intentionally set to null, then filled using the mode (most frequent value). New derived columns are created from the order date including month, quarter, and day of week. A separate filtered dataframe is maintained containing only completed orders, which is used for all revenue-based calculations to avoid counting returned or cancelled orders.

Step 3 — Exploratory Data Analysis
Summary statistics are computed using Pandas groupby operations. Revenue is aggregated by category to identify top-performing segments. Order status distribution is calculated to understand the ratio of completed, returned, and cancelled orders. Average order value is computed per sales channel to identify the most profitable acquisition source.

Step 4 — Data Visualisation (Matplotlib & Seaborn)
Ten charts are generated and saved as PNG files covering the following analyses:

Monthly Revenue Trend — A line chart with a 3-month rolling average to identify seasonal patterns and growth trajectory across 2023.
Revenue by Category — A horizontal bar chart comparing total revenue across all five product categories.
Sales Channel Performance — Side-by-side bar charts comparing total revenue and average order value across channels like Organic Search, Paid Ads, Social Media, Email, and Direct.
Customer Segment Heatmap — A Seaborn heatmap crossing age group against gender to identify which customer segments generate the highest revenue.
Top 10 Products — A horizontal bar chart colour-coded by category showing the highest revenue-generating individual products.
Order Status Donut Chart — A donut chart showing the percentage split between completed, returned, and cancelled orders with total order count at the centre.
Regional Quarterly Revenue — A grouped bar chart showing how revenue varies across five regions over four quarters.
Price vs Revenue Scatter Plot — A scatter plot visualising the relationship between unit price and order revenue, with each category in a different colour.
Day-of-Week Order Volume — A bar chart showing order frequency across weekdays and weekends, with weekends highlighted separately.
Discount Impact Analysis — A dual-axis chart combining bars for order volume and a line for average revenue at each discount level to understand the effect of discounting on sales behaviour.

Step 5 — Power BI Export
A summarised CSV file is generated by aggregating the cleaned data across month, category, region, and channel. This pre-aggregated file is optimised for direct import into Power BI to build interactive dashboards for stakeholder reporting and trend analysis.

Tech Stack

NumPy — random data generation and numerical operations
Pandas — data loading, cleaning, transformation, and aggregation
Matplotlib — chart creation, formatting, and export
Seaborn — statistical visualisations including heatmaps
Power BI — interactive dashboard and business reporting layer
